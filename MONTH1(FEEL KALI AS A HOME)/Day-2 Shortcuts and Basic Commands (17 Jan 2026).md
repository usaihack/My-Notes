
### **Some Basic but useful Shortcuts for Productivity**:

| ctrl + shift + R | Split Terminal                        |
| ---------------- | ------------------------------------- |
| ctrl + shift + E | Close split terminal                  |
| ctrl + shift + T | New Terminal Tab                      |
| ctrl + A         | Move cursor to start of the command   |
| ctrl + E         | Move cursor to the End of the command |
| ctrl + K         | Delete command from start to end      |
| ctrl + U         | Delete command from end to start      |
| ctrl + L         | clears the Terminal or use `clear`    |


### **Some Basic but useful Commands:**

| `pwd`                               | current working directory                                                                                                                                                                                                                                                                                                 |
| ----------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ls`                                | list all the files and directories                                                                                                                                                                                                                                                                                        |
| `ls -a`                             | list all the files and directories along with hidden ones                                                                                                                                                                                                                                                                 |
| `ls -l`                             | list files and directories with some details                                                                                                                                                                                                                                                                              |
| `mkdir`                             | create directory. Use the flag `-p` so that it will exit silently if the directory exists or it will create it. Pros always use it ðŸ˜‰.                                                                                                                                                                                    |
| `rmdir`                             | delete empty directory                                                                                                                                                                                                                                                                                                    |
| `touch fileName`                    | create empty file                                                                                                                                                                                                                                                                                                         |
| `rm`                                | deletes files                                                                                                                                                                                                                                                                                                             |
| `rm -rf directoryName`              | when directory is not empty (use with care)                                                                                                                                                                                                                                                                               |
| `nano fileName`                     | GUI based file editor                                                                                                                                                                                                                                                                                                     |
| `echo > file.txt`                   | write to the file in overwritten mode                                                                                                                                                                                                                                                                                     |
| `echo >> file.txt`                  | write to the file in appended mode                                                                                                                                                                                                                                                                                        |
| `command1 && command2 && ....`      | `&&` is used to chain commands in a single. If first command was unsuccessful, the rest will be stopped or if second and so on...                                                                                                                                                                                         |
| pipe \|                             | pipe is used to make the output of the first command as an input to the second command and so on...<br>**Syntax:**<br>command1 \| command2 \| ...<br><br>**Example:**<br>`ls` \| `grep txt`<br><br>`grep` will take as an input whatever is the output of `ls`, search for txt containing text and will display only that |
| `cat`                               | read content from the files                                                                                                                                                                                                                                                                                               |
| `less`                              | read big content page by page                                                                                                                                                                                                                                                                                             |
| `head`                              | read first 10 lines only                                                                                                                                                                                                                                                                                                  |
| `tail`                              | read last 10 lines only                                                                                                                                                                                                                                                                                                   |
| `cp originalFileName copyFileName`  | makes a copy of the original file                                                                                                                                                                                                                                                                                         |
| `cp -r originalFolder copiedFolder` | makes a copy of the original Folder, `-r` for folders                                                                                                                                                                                                                                                                     |
| `mv`                                | Two use cases:<br>1. For moving a file from one folder to another<br>`mv fileName Folder/`<br><br>2. For renaming<br>`mv oldFileName newFileName`<br><br>**OR**<br><br>move and rename together:<br>`mv oldFileName Folder/newFileName`                                                                                   |
